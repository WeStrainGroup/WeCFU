# WeCFU 使用说明（中文详细版）

适用版本：v1.0.7　|　适用系统：macOS / Linux / Windows

---

## 0. 一句话总览

一个浏览器里跑的菌落计数工具：把照片拖进去 → 自动数 → 你点几下校正 → 导出 CSV 和带标注的图。无需联网（网页版除外），数据全在本地。

---

## 1. 三种用法，挑一个

### 1.1 网页版（最快，零安装）

浏览器直接打开 **https://huggingface.co/spaces/WeCFU/wecfu**，把图片拖进去就能用。
单次会话上限 100 张图 / 400 MB，1 小时不动会自动清空。适合临时、偶尔、给别人演示。

### 1.2 本地 conda（长期 / 大批量 / 离线推荐）

```bash
conda install westraingroup::wecfu
wecfu serve
```

`wecfu serve` 跑起来后浏览器自动开 http://127.0.0.1:8765 。没有图片数量限制，完全离线。

**Windows 用户**：命令完全一样，只是要在开始菜单的 **「Anaconda Prompt」** 里跑（不是 cmd / PowerShell）。需要先装 [Miniconda for Windows](https://docs.conda.io/projects/miniconda/en/latest/)。

> 想要全新独立环境（不污染 base）：
> `conda create -n wecfu -c westraingroup -c conda-forge wecfu -y && conda activate wecfu`

### 1.3 命令行批跑（无界面）

```bash
wecfu batch /路径/到/照片文件夹 --out /路径/到/输出目录
```

直接产出 `results.csv` + 每张图的标注 PNG，适合脚本化大批量处理。

## 2. 从导入到导出（GUI 完整流程）

### 2.1 导入图片

- **拖拽（推荐）**：把图片文件或整个文件夹拖到左侧拖拽区。拖文件夹时顶部会显示读取进度和上传百分比。
- **粘贴路径**（仅本地版）：在左侧输入框粘绝对路径，点「Ingest path」。只建符号链接、不复制原图；Windows 上若不允许符号链接会自动改成复制，同样不动原图。

> 原图**永远不会被修改**。

### 2.2 计数

两种方式，效果一样：

- 点顶部 **「Count all」** → 一次性数完所有图。
- 或者**直接点开某一张还没数的图** → 自动数这一张（顶部状态栏会显示 "Auto counting this plate…"）。

机器识别到的菌落用 **蓝色圆圈 + 编号** 标出。

### 2.3 人工校对

点左侧任意图片打开，然后：

| 你想做的事 | 操作 |
|---|---|
| 加一个漏识别的菌落 | **左键**点空白皿面（画成绿色圈） |
| 删掉一个误识别 | **右键**点那个圈 |
| 撤销上一步增 / 删 | **Cmd-Z / Ctrl-Z** 或点「↶ Undo」 |
| 放大 / 缩小 | 鼠标**滚轮**（以光标为中心） |
| 平移 | **Cmd / Ctrl + 拖拽** |
| 上一张 / 下一张 | **← / →** 方向键 |

**没有"标记已复核"按钮了**——只要你点开过一张图，它在左侧的小圆点就会自动变绿，表示"你看过了"。

### 2.4 左侧小圆点的三种颜色

| 颜色 | 含义 |
|---|---|
| 灰色 | 已上传，还没数 |
| 蓝色 | 机器数过了，你还没点开看 |
| 绿色 | 你已经查看或修改过 |

（圆圈和圆点配色统一：**蓝=机器**，**绿=人工 / 已看**。）

## 3. 调参（适配不同菌落 / 培养基）

参数栏分三行：
- **第 1 行**：预设下拉 + 「Apply and recount」按钮
- **第 2 行**：6 个算法参数（Min value / Max saturation / Plate inset / Min circularity / Min area frac / Peak min distance）
- **第 3 行**：New colony radius + Live threshold preview（这两个是辅助项，不影响算法）

**预设**下拉一键切换四种常见情形：

| 预设 | 适用 |
|---|---|
| White colonies（默认） | YM 1× 上的白 / 奶白菌落 |
| Cream or pale yellow | 偏黄但仍亮 |
| Yellow colonies | 明显黄色菌落 |
| Any bright | 兜底，会带噪 |

精调时勾上 **「Live threshold preview」**：移动 Min value / Max saturation 时，画面里被算作菌落的像素会**实时绿色高亮**，所见即所得。调满意后点 **「Apply and recount」** 只对当前这张图重数。

> 每次重数，CSV 里的 `machine_count` 会跟着更新为最新算法的结果。

---

## 4. 参数详解（什么时候往哪边调）

默认值已经对 YM 1× 白菌落场景调过，多数情况不用动。

| 参数 | 范围 / 默认 | 漏识真菌落 → | 误识背景 → |
|---|---|---|---|
| **Min value**（亮度下限，HSV 的 V） | 0–255 / 200 | 调低（180、160） | 调高（215、225） |
| **Max saturation**（饱和度上限，HSV 的 S；越低越严格选白色） | 0–255 / 60 | 想纳入黄/橙菌落就调高（120、200） | 黄色培养基串色被识别就调低（40、30） |
| **Plate inset**（保留皿半径的比例） | 0.5–1.0 / 0.82 | 皿壁附近菌落被漏 → 调高（0.88） | 皿壁反光被识别 → 调低（0.78） |
| **Min circularity**（圆度下限） | 0–1 / 0.55 | 不规则菌落被滤掉 → 调低（0.4） | 非圆脏东西被识别 → 调高（0.7） |
| **Min area frac**（最小面积占皿比例） | / 0.0001 | 小菌点被漏 → 调低（0.00005） | 噪点被识别 → 调高（0.0003） |
| **Peak min distance**（拆分相邻菌落的最小种子间距，像素） | / 15 | 两个粘连菌落被当成一个 → 调低（8） | 一个菌落被劈成两半 → 调高（25） |

**New colony radius**：你左键加菌落时画的绿圈大小，纯视觉，不影响计数。

## 5. 难板子怎么处理

- **密集 / 粘连**：调低 Peak min distance（比如 8），点「Apply and recount」。
- **铺满 / 菌膜**：算法基本数不准。建议手动估算密度（OD600 或传代再数），并在备注里写"菌膜不可数"。
- **黄色蔓延 / swarming**：切到「Yellow colonies」预设试试。
- **空盘出现 1–2 个误识圈**：右键删掉，或调高 Min value（215+）。
- **真菌落漏了**：左键点空白处直接加（最快），或调低 Min value 后重数。

## 6. 文件名规则（可选，但批量分析推荐）

工具会自动从文件名解析元信息（仅用于你自己归档，不进精简版 CSV）。建议格式：

```
PXXVNN_G[+-]_<medium>_<dilution>_<aer|ana>_dayN_<rep>_<YYYYMMDD_HHMMSS>.jpg
```

例如：`P01V18_G+_YM_1X_aer_day9_1_20260521_162517.jpg`

文件名不符合规则也能正常计数。

## 7. 导出

顶部两个按钮，都会弹出对话框让你**自定义文件名**（默认与图片名一致）。在 Chrome / Edge 上还能选保存到哪个文件夹；Safari / 网页版走浏览器默认下载位置。

### 7.1 「Export CSV」——只要计数表

5 列，只保留你最关心的信息：

| 列 | 含义 |
|---|---|
| `filename` | 图片文件名 |
| `total_count` | 最终菌落总数（人工增删之后） |
| `machine_count` | 算法最近一次数出来的数量 |
| `n_removed` | 人工删掉的机器识别数 |
| `n_added` | 人工新增的菌落数 |
| `notes` | 你给这张图写的备注 |

关系：`total_count = machine_count − n_removed + n_added`。

### 7.2 「Export bundle」——完整打包 zip

包含上面那个 CSV + 每张图的标注 PNG（已裁到皿、灰底，可直接放论文 SI）+ 每张图的 JSON 状态档（含每个菌落坐标、来源、参数、备注；想完全复现这次计数就靠它）。

## 8. 常见问题

**Q：重启 WeCFU 之后，上次导入的图片还在吗？**
A：不在。每次启动 `wecfu serve` 都会清空上次的导入和计数痕迹，保证干净工作台。想留进度就导出 zip。

**Q：plate（皿）没找对怎么办？**
A：极少数情况下圆找错。把 Plate inset 调到 0.7 让识别圈缩进避开错误；顶多手动增删几个圈。

**Q：中文 / 含空格的路径能用吗？**
A：能。读图用的是兼容非 ASCII 路径的方式（这点在 Windows 上尤其重要）。

**Q：网页版多人同时用会互相看到吗？**
A：不会。每个访客有独立会话，互相隔离，1 小时无活动自动清除。

## 9. 关键词速查

- **machine（蓝圈 / 蓝点）**：算法自动识别的菌落
- **manual（绿圈）**：你左键手动加的菌落
- **绿点（左侧）**：你已经查看 / 修改过这张图
- **Live threshold preview**：实时把"被认作菌落"的像素绿色高亮，用于调参

## 10. 反馈与改进

- `plate.py` — 皿定位
- `segment.py` — 计数算法 / 预设
- `overlay.py` — 导出图样式
- `server/app.py` — 后端接口（本地 + 网页两种模式）
- `server/static/app.js` — 前端交互

开发模式下 `pip install -e .` 不用重装，重启 `wecfu serve` 即生效。仓库：github.com/WeStrainGroup/WeCFU 。
