# CFU counter 使用说明（中文详细版）

适用版本：v0.1
适用系统：macOS / Linux（Windows 应该也行，但未测）

---

## 0. 一句话总览

一个浏览器里跑的菌落计数工具：把照片拖进去 → 自动数 → 你点几下校正 → 导出 CSV 和带标注的图。

---

## 1. 安装

只要装一次。

```bash
# 1) 进 cfu-counter 项目目录
cd ~/claude_code_workspace/WeF/cfu-counter

# 2) 创建 conda 环境（约 3–5 分钟，下载 ~500 MB）
conda env create -f environment.yml

# 3) 激活并安装本工具
conda activate cfu-counter
pip install -e .
```

验证一下能跑：

```bash
cfu-counter --help
```

应该看到 `batch / serve / sam` 三个子命令。

---

## 2. 启动 GUI

```bash
conda activate cfu-counter
cfu-counter serve
```

终端会显示 `cfu-counter serving at http://127.0.0.1:8765`，浏览器会自动打开这个地址（如果没自动开就手动复制粘贴）。

不想自动开浏览器，加 `--no-open`；换端口 `--port 9000`。

---

## 3. 第一次跑：从导入到导出

### 3.1 导入图片

**方式 A：拖拽（最快）**
把图片文件或整个文件夹拖到左上角"把图片或文件夹拖到这里"区域。

**方式 B：粘贴路径（推荐用于已有文件夹）**
在左上输入框里粘贴绝对路径（例如 `/Users/你/Documents/.../照片/拍菌/260521`），点"导入路径"。这种方式只建**符号链接**，**绝不会复制或修改你的原图**。

每次导入都会生成一个**批次（batch）**，左侧批次下拉里能切换。

### 3.2 自动计数

点顶部的 **「全部计数」**。
工具会用默认参数（白色菌落）对这个批次里所有未处理的图运行 Layer 1。
完成后左侧每张图右侧会显示菌落数。

### 3.3 校对单张

点左侧任一张图 → 主画面会显示这张图，每个识别到的菌落都画一个绿圈+编号。

操作：

| 你想做的事 | 操作 |
|---|---|
| 加一个漏识别的菌落 | **左键单击**空白处 |
| 删除一个误识别 | **右键单击**那个圈 |
| 撤销刚才的一步 | **Cmd-Z / Ctrl-Z**，或点"↶ 撤销"按钮 |
| 放大 / 缩小 | 鼠标**滚轮**（以光标为中心） |
| 平移 | **Cmd / Ctrl + 鼠标拖拽** |
| 切换上 / 下一张 | **← / →** 方向键 |
| 标记此图已复核 | **空格**，或点"标记已复核"按钮 |

新增菌落的默认半径在右下参数栏 **「新增半径」** 里调，默认 40 px——对你目前 ~2000 px 宽的图比较合适。

### 3.4 改参数 / 适配不同培养条件

参数栏有 **「预设」** 下拉：

| 预设 | 适用 | 改了什么 |
|---|---|---|
| 白菌落（默认） | YM 1× 上的白/奶白菌落 | min_value 200, max_sat 60, circ 0.55 |
| 乳白/淡黄 | 偏黄但仍亮的菌 | min_value 180, max_sat 100 |
| 黄色菌落 | 明显黄色菌落 | min_value 160, max_sat 200 |
| 任意亮色 | 兜底，会带噪 | min_value 140, max_sat 255, circ 0.45 |
| 自定义 | 你自己改了某个数 | — |

勾选 **「实时阈值预览」**，移动 `min_value` / `max_saturation` 时，画面里会实时**绿色高亮**所有"被认作菌落像素"的区域。这是调参最直观的方式——你能立刻看到把谁纳入、把谁排除。

调完后点 **「应用并重跑」**（或等同的"重新计数"），只对当前这张图重跑。

## 4. 参数详解（什么时候改什么）

**默认值已经对 YM 1× 白菌落场景调过，多数情况不用动。** 下面是参数含义和典型误差表现：

### `min_value` (0–255)  — 默认 200
HSV 的 V 通道。**像素必须比这个值更亮才算菌落。**
- 真实菌落漏了 → 调**低**（180、160）
- 培养基反光被误识 → 调**高**（215、225）

### `max_saturation` (0–255) — 默认 60
HSV 的 S 通道。**像素必须比这个值更"接近白色"。** 黄菌落 S 高，白菌落 S 低。
- 想包含黄色/橙色菌落 → 调**高**（120、200）
- 黄色培养基串色被误识 → 调**低**（40、30）

### `plate_inset` (0.5–1.0) — 默认 0.82
**裁的圆的半径占皿半径的多少。** 0.82 表示只保留中心 82% 半径，外围 18% 全部丢弃，用来避免皿壁反光。
- 皿壁附近有真菌落被漏 → 调**高**（0.88、0.92）
- 皿壁反光被误识 → 调**低**（0.78、0.75）

### `min_circularity` (0–1) — 默认 0.55
**轮廓圆度下限**。完美的圆 = 1，长条 = 0。
- 不规则边的菌落（粗糙、分裂）被滤掉 → 调**低**（0.40、0.30）
- 一坨非圆形脏东西被误识 → 调**高**（0.70）

### `min_area_frac` — 默认 0.0001
**最小菌落面积，作为皿面积的占比。**
- 很小的菌点被漏 → 调**低**（0.00005、0.00002）
- 噪点被误识 → 调**高**（0.0003）

### `peak_min_distance` (px) — 默认 15
**两个相邻菌落中心至少要差多少像素**才会被拆成两个（分水岭种子距离）。
- 相邻两个菌落被当成一个 → 调**低**（8、5）
- 一个菌落被拆成两个半月 → 调**高**（25、40）

### 「新增半径」
你**左键加菌**时画的圆有多大。仅影响显示和导出图里的圆大小，不影响计数对错（一个新增 = 一个 CFU）。

## 5. 难板子怎么处理

### 5.1 密集/粘连（很多菌挤在一起）
1. 调低 `peak_min_distance`（比如 8）+ 点"重新计数"。
2. 还是不行，点 **「SAM」** 按钮（第一次会提示装权重和依赖，约 358 MB+2 GB，**只装一次**）。SAM CPU 大约 30 秒/张。

### 5.2 铺满/菌膜
基本数不准，工具会把整个菌膜识别成 0–少量大块。建议：
- 手动估算菌落密度（OD600 或后续传代再数）
- 在 GUI 里左键标几个代表性点，备注里写"菌膜，不可数"

### 5.3 黄色蔓延/swarming
切到"黄色菌落"预设，看看能不能勾出菌斑边缘。若不能，按上面处理。

### 5.4 空盘上出现 1–2 个圈
通常是培养基的黄褐色串色或刮痕。
- 右键删掉
- 或调高 `min_value`（215+）让阈值更严

### 5.5 真菌落漏了
- 左键点空白处加（最快）
- 或调低 `min_value` 然后重跑

## 6. 文件名规则与元数据

工具自动从文件名解析以下字段并写进 CSV：

```
PXXVNN_G[+-]_<medium>_<dilution>_<aer|ana>_dayN_<rep>_<YYYYMMDD_HHMMSS>.jpg
```

例如：`P01V18_G+_YM_1X_aer_day9_1_20260521_162517.jpg`

解析出：
- `plate=P01V18` （样本和版本）
- `gram=G+`
- `medium=YM`
- `dilution=1X`
- `atmo=aer`
- `day=9`
- `rep=1`
- `timestamp=2026-05-21T16:25:17`

文件名不符合规则也能数，只是 CSV 元数据列为空。

## 7. 导出

### 7.1 仅 CSV
顶部 **「导出 CSV」** → 下载 `cfu_<批次>.csv`。

字段：filename / plate / gram / medium / dilution / atmo / day / rep / timestamp / **cfu_count** / mean_radius_px / plate_radius_px / reviewed / method / n_manual / low_confidence / notes

### 7.2 完整打包（推荐发给协作者 / 留底）
顶部 **「导出打包」** → 下载 `cfu_<批次>.zip`，包含：
- `cfu_<批次>.csv`
- `overlays/*.png`：每张图带标注的复核图（已裁到皿+灰色背景，可直接放论文 SI）
- `detections/*.json`：每张图的"状态档案"（含每个菌落坐标、半径、来源、参数、备注、是否人工复核）。如果以后想完全复现这次计数，把 JSON 喂回 `cfu-counter` 就行。

---

## 8. 命令行（无界面批跑）

完全跳过 GUI，直接产 CSV：

```bash
cfu-counter batch /path/to/photos --out /path/to/run_dir
```

输出：
- `run_dir/results.csv`
- `run_dir/overlays/*.png`
- `run_dir/detections/*.json`

加 `--force` 强制重新计数（默认会跳过已处理的图）。

## 9. 常见问题

**Q：导入时报"image not found"？**
A：路径要绝对（以 `/` 开头）。中文/空格路径可以，但最好直接拖拽进去更稳。

**Q：plate 没找对？**
A：极少数情况下 Hough 会找错圆。表现是叠加图上灰圈不在皿上。临时方案：在 GUI 把 `plate_inset` 调到 0.7 让圈缩进里面避开错误；长期想修可以提个 issue（plate.py:detect_plate）。

**Q：我手动改完了，再点"全部计数"会不会覆盖？**
A：不会。只要状态里 `reviewed=True`（你点过"标记已复核"或动过菌圈），批跑就跳过这张。除非加 `--force`。

**Q：我能换台机器继续吗？**
A：可以。把 `data/inputs/<批次>/` 和 `data/runs/<批次>/detections/` 拷过去（注意原图路径要么真存在要么改成上传），重启 `cfu-counter serve` 即可。

**Q：图很多很卡？**
A：单 worker 默认。你可以多开几个 batch 分批处理；或者用 `cfu-counter batch ...` 命令行多核场景。

## 10. 关键词速查

- **批次（batch）**：一次导入对应的目录，对应 `data/inputs/<batch>/`
- **detection**：一个被识别到的菌落（cx, cy, r, source）
- **source**：cv = 经典 CV / sam = SAM / manual = 你左键加的
- **reviewed**：你确认过这张图，下次批跑跳过
- **low_confidence**：算法觉得这板"可能数不准"（铺满 / 大量异形）
- **mask preview**：实时把"被认作菌落"的像素用绿色高亮
